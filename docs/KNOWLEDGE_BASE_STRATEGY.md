# KNOWLEDGE_BASE_STRATEGY.md

## Objetivo
Criar uma base confiável, versionada e útil para suporte, clientes e IA.

## Tipos de conteúdo
- Artigos públicos.
- Artigos internos.
- Playbooks.
- Troubleshooting.
- Procedimentos operacionais.
- Integrações.
- Causas conhecidas.
- Scripts de atendimento.

## Regras
- Todo artigo deve ter status.
- Todo artigo publicado deve ter revisão versionada.
- Conteúdo interno não pode ser exposto no portal público.
- Conteúdo sensível deve ser classificado antes de indexação.
- IA só pode usar conteúdo aprovado.

## Status sugeridos
- draft
- review
- approved
- published
- archived

## Migração de raw_knowledge
Antes de migrar:
1. Inventariar arquivos.
2. Classificar sensibilidade.
3. Separar público, interno, sensível e descartável.
4. Normalizar conteúdo.
5. Criar revisões.
6. Só depois indexar para busca/IA.
