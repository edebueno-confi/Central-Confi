# ENGINEERING_WORKFLOW.md

## Objetivo
Criar uma ponte clara entre suporte e tecnologia sem transformar tickets em backlog técnico bagunçado.

## Work items
Tipos:
- bug
- improvement
- technical_task
- investigation

## Fluxo
1. Suporte identifica necessidade técnica.
2. Cria ou solicita work item vinculado ao ticket.
3. Engenharia triageia.
4. Define status técnico.
5. Atualiza progresso.
6. Suporte recebe devolutiva traduzível para cliente.
7. Ticket é atualizado sem misturar conversa técnica com conversa pública.

## Status técnico sugerido
- triage
- accepted
- rejected
- in_progress
- waiting_external
- released
- cancelled

## Regras
- Um ticket pode vincular vários work items.
- Um work item pode impactar vários tickets.
- Work item não substitui ticket.
- Ticket não substitui backlog técnico.
- Devolutiva técnica deve ser rastreável.
